创建一个Vue应用

应用示例

每个Vue应用同时通过createApp函数创建一个新的应用示例，也就是从某种意义上，vue是一个面向组件的语言

import {createApp} from 'vue'
const app=createApp{{
    /*根组件选项*/ 
}}

根组件

每个vue应用都需要一个”根组件”，其他组件将作为其子组件。在大多数情况下，应用都是由一颗嵌套的、可重复的组件树组成的。比如对应组建的表现形式：

App (root component)
├─ TodoList
│  └─ TodoItem
│     ├─ TodoDeleteButton
│     └─ TodoEditButton
└─ TodoFooter
   ├─ TodoClearButton
   └─ TodoStatistics

在后续我们将会学习如何定义和组合多个组件。下面我们将关注一个组件内部发生了什么。

挂载应用

应用实例必须在调用了.mount()方法后才会渲染出来。该方法接受相应的容器参数，可以是一个实际的DOM元素后世一个css选择器的字符串。

<div id="app"></div>

app.mount('#app')

应用根组件的内容将会被渲染在容器元素里面。容器元素自己将不会被视为应用的一部分。mount()方法应该始终在整个应用配置和资源注册完成后被调用，对应的返回值是根组件实例而非应用实例。

DOM中的根组件模板

在未采用构建流程的情况下适用vue时，我们可以挂载容器中直接书写根组件模板。

<div id="app">
  <button @click="count++">{{ count }}</button>
</div>

import { createApp } from 'vue'
const app = createApp({
  data() {
    return {
      count: 0
    }
  }
})
app.mount('#app')

当根组件没有设置template选项时，Vue将自动适用容器的innerHTML作为模板。

应用配置

应用实例会提供一个.config对象允许我们配置一些应用级的选项，例如定义一个应用级别的错误处理器，将捕获所有由子组件上抛未被处理的任务。

app.config.errorHandler = (err) => {
  /* 处理错误 */
}

应用实例还提供了一些方法来注销应用范围内可用的资源，例如注销一个组件：

app.component("TodoDeleteButton",TodoDeleteButton)

这使得我们可以在任何地方使用TodoDeleteButton，从而实现对组件的注销。

多个应用实例

应用实例并不只限于一个，createApp API允许在同一个页面上创建多个共存的vue应用，并且每个应用都拥有自己的用于配置和全局资源的作用域。

模板语法

Vue是一种基于HTML的模板语法，使我们能够声明式的将组件实例的数据绑定到呈现的DOM上。所有的vue模板都是语法层面合法的HTML，可以被符合规范的浏览器和HTML解释器解析。
在底层机制中，Vue会将模板编译成高度优化的JavaScript代码。结合响应式系统，当应用状态变更时，Vue能够智能的推导出需要重新渲染的组建的最少数量，并应用最少的DOM操作。

文本插值

最基本的数据板顶方式是文本插值，它使用的是”Mustance”语法(双大括号语法)。

<span>Message: {{ msg }}</span>

其中大括号中的值会随着msg的变换而变换。同时，对应的msg的属性变化的时候他也会同步更新。

原始Html

双大括号会使得对应的数据插值为纯文本，而不是HTML。如果想要插入HTML，需要使用v-html

<p>Using text interpolation: {{ rawHtml }}</p>
<p>Using v-html directive: <span v-html="rawHtml"></span></p>

v-html为一个指令，指令由v-作为前缀，表明他们是一些由vue提供的一些特殊指令。需要注意的是，vue不是一个基于字符串的模板引擎。在使用vue时，应当使用组件作为UI重用和组合的基本单元。

Attribute绑定

双大括号不能在HTML attributes 中使用。想要响应式的绑定一个attribute,需要使用v-bind命令。

<div v-bind:id="dynamicId"></div>

v-bind指令指示vue将元素的id attribute与组件的dynamicId属性保持一致。如果绑定的值是null``或者undefined，那么该attribute将会从渲染的元素上移除。 因为v-bind```非常常用，所以我们提供了特定的简写语法。

<div :id="dynamicId"></div>

布尔型 Attribute

布尔型 attribute 依据true / false的值来决定attribute是否存在于该元素上。正如disabled而言

<button :disabled="isButtonDisabled">Button</button>

当isButtonDisabled为真值或一个空的字符串的时，元素会包含一个disabledattribute。而当其为其他假值时，attribute将被忽略。

动态绑定多个值

如果某个JavaScript对象有多个Attribute时，此时，我们在板顶元素的时候，可以不需要指定具体的某个属性。

const objectOfAttrs = {
  id: 'container',
  class: 'wrapper'
}

通过不带参数的v-bind,你可以将他们绑定在单个元素上：

<div v-bind="objectOfAttrs"></div>

使用JavaScript表达式

我们仅在模板中绑定了一些简单的属性名。但是vue实际上在所有的数据板顶中支持完整的JavaScript表达式：

{{ number + 1 }}

{{ ok ? 'YES' : 'NO' }}

{{ message.split('').reverse().join('') }}

<div :id="`list-${id}`"></div>

这些表达式都会被作为JavaScript，以组件为作用域解析执行。
在vue模板内，JavaScript表达式可以被使用在如下场景上：
·在文本插值中
·在任何vue指令(以v-开头的特殊attribute)attribute的值中
需要注意的是，仅仅支持单一的表达式，因而，对于如下的情况下是不可以的!!!

<!-- 这是一个语句，而非表达式 -->
{{ var a = 1 }}

<!-- 条件控制也不支持，请使用三元表达式 -->
{{ if (ok) { return message } }}

调用函数

可以在绑定的表达式中使用一个组件暴露的方法

<span :title="toTitleDate(date)">
  {{ formatDate(date) }}
</span>

需要考虑到，绑定表达式中的方法在组件每次更新时都会被重新调用，因此不应该产生任何副作用。

受限的全局访问

模板中表达式将被沙盒化，仅能访问到有效的全局对象列表。

指令 Directives

指令是带有v-前缀的特殊attribute。vue提供许多内置指令，包括上面我们所介绍的v-bind和v-html
指令attribute的期望值为一个JavaScript表示式(除了少数几个例外，即之后要讨论到的v-for、v-on和v-slot)。一个指令的任务是在其表达式的值变换的时候响应式的更新DOM，以v-if为例：

<p v-if="seen">Now you see me</p>

这里，v-if指令会根据表达式seen的值的真假来移除/插入该

元素。

参数Arguments

某些指令会需要一个”参数”，在指令名后通过一个冒号隔开做标识，例如用v-bind指令来响应式地更新HTML attribute:

<a v-bind:href="url"> ... </a>

<!-- 简写 -->
<a :href="url"> ... </a>

这里href就是一个参数，他告诉v-bind``指令将表达式url的值绑定到元素的href attribute上。在简写中，参数前的一切都会被缩略为一个:字符。 另一个例子是v-on```指令，他将会监听DOM事件：

<a v-on:click="doSomething"> ... </a>

<!-- 简写 -->
<a @click="doSomething"> ... </a>

这里的参数主要是用来监听事件的名称：clicked。v-on有一个相应的缩写，即@字符。我们之后也会讨论关于事件处理的更多细节。

动态参数

同样在指令参数上也可以使用一个JavaScript表达式，需要包含在一对方括号内：

<!--
注意，参数表达式有一些约束，
参见下面“动态参数值的限制”与“动态参数语法的限制”章节的解释
-->
<a v-bind:[attributeName]="url"> ... </a>

<!-- 简写 -->
<a :[attributeName]="url"> ... </a>

这里的attributeName会作为一个JavaScript表达式被动态执行，计算得到的值会被用作最终的参数。
相似的，你还可以将一个函数绑定到动态的事件名称上

<a v-on:[eventName]="doSomething"> ... </a>

<!-- 简写 -->
<a @[eventName]="doSomething">

在此示例中，当eventName的值是"focus"，v-on:[eventName]就等价于v-on:focus

动态参数的限制：

动态参数的表达式的值应当是一个字符串，或者是null。特殊值```null``意为移除该绑定。其他的非字符串的值将会触发警告。

动态参数语法的限制

动态参数的语法因为某些字符的缘故有一些语法限制，比如空格和引号，在HTML attribute名称中都是不合法的。

<!-- 这会触发一个编译器警告 -->
<a :['foo' + bar]="value"> ... </a>
